# Faith App — Phase 1: Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the Expo/React Native project with Firebase Auth (Sign in with Apple + Google), a Firestore data model with tested security rules, a Bible API client, and a navigation shell that switches between a sign-in screen and a home screen based on auth state — the foundation every later feature (Phases 2–4) builds on.

**Architecture:** A single Expo-managed React Native app (TypeScript) using the Firebase JS SDK directly (not `@react-native-firebase`) so it keeps running in Expo Go without a custom dev client. Auth state drives navigation via `onAuthStateChanged`; Firestore access is scoped per-user by security rules tested against the Firebase Local Emulator Suite; Bible verse text is fetched at runtime, never bundled.

**Tech Stack:** Expo (managed workflow) + TypeScript, Firebase JS SDK (`firebase` npm package: Auth + Firestore), `expo-apple-authentication`, `expo-auth-session` (Google OAuth), `@react-navigation/native` + native-stack, Jest (`jest-expo` preset) + `@testing-library/react-native`, `@firebase/rules-unit-testing` + Firebase Local Emulator Suite for security rules tests.

## Global Constraints

- Platform: React Native via Expo managed workflow, targeting both iOS and Android from one codebase — do not eject or add custom native modules that require a dev client in this phase.
- Auth: Firebase Authentication using the Firebase JS SDK (`firebase` package), with Sign in with Apple and Sign in with Google as the only providers.
- Data: Firebase Firestore only — no custom backend server is built or hosted.
- Bible content: fetched at runtime from api.bible (scripture.api.bible); never bundled in the app. (This finalizes the Bible API vendor choice left open in the spec's Section 10.)
- Language: TypeScript throughout.
- Testing: Jest with the `jest-expo` preset and `@testing-library/react-native` for component/unit tests; Firestore security rules are tested against the Firebase Local Emulator Suite via `@firebase/rules-unit-testing` — rules are never considered done until they pass emulator tests.
- All Firebase/Google/Bible API credentials are read from environment variables (`EXPO_PUBLIC_*` prefix for client-exposed values) and must never be committed to git.

## Prerequisites (do once, before Task 1)

These are manual account/console setup steps that can't be scripted — do them before starting Task 1:

1. In the [Firebase console](https://console.firebase.google.com), create a project (e.g. "faith-app"). Enable **Firestore** (start in production mode — rules are added in Task 2) and enable **Authentication** with the **Apple** and **Google** sign-in providers.
2. In that Firebase project, register an iOS app and an Android app (Project Settings → Your apps). Note the six web config values shown (`apiKey`, `authDomain`, `projectId`, `storageBucket`, `messagingSenderId`, `appId`) — these go into `.env` in Task 2.
3. Create a free account at [api.bible](https://scripture.api.bible) and generate an API key — needed for Task 3.
4. In the [Google Cloud Console](https://console.cloud.google.com) for the same Firebase project, create an OAuth 2.0 Client ID of type **Web application** (used by `expo-auth-session` via Expo's auth proxy during development) — needed for Task 5.

```typescript
export class BibleApiError extends Error {}

const TRANSLATION_BIBLE_IDS: Record<string, string> = {
  KJV: 'de4e12af7f28f599-02',
  WEB: '9879dbb7cfe39e4d-04',
};

const BASE_URL = 'https://api.scripture.api.bible/v1';

export async function fetchVerseText(
  reference: string,
  translation: string,
  apiKey: string
): Promise<string> {
  const bibleId = TRANSLATION_BIBLE_IDS[translation];
  if (!bibleId) {
    throw new BibleApiError(`Unsupported translation: ${translation}`);
  }

  const response = await fetch(
    `${BASE_URL}/bibles/${bibleId}/search?query=${encodeURIComponent(reference)}`,
    { headers: { 'api-key': apiKey } }
  );

  if (!response.ok) {
    throw new BibleApiError(`Bible API request failed with status ${response.status}`);
  }

  const data = await response.json();
  const passage = data?.data?.passages?.[0]?.content;
  if (!passage) {
    throw new BibleApiError(`No passage found for reference: ${reference}`);
  }

  return passage;
}
```

```typescript
import * as AppleAuthentication from 'expo-apple-authentication';
import { OAuthProvider, signInWithCredential, UserCredential } from 'firebase/auth';
import { auth } from './config';

export class AppleSignInError extends Error {}

export async function signInWithApple(): Promise<UserCredential> {
  const credential = await AppleAuthentication.signInAsync({
    requestedScopes: [
      AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
      AppleAuthentication.AppleAuthenticationScope.EMAIL,
    ],
  });

  if (!credential.identityToken) {
    throw new AppleSignInError('Apple sign-in did not return an identity token');
  }

  const provider = new OAuthProvider('apple.com');
  const firebaseCredential = provider.credential({
    idToken: credential.identityToken,
  });

  return signInWithCredential(auth, firebaseCredential);
}
```

Create `app/src/hooks/useGoogleSignIn.ts`:
```typescript
import { useEffect } from 'react';
import * as Google from 'expo-auth-session/providers/google';
import * as WebBrowser from 'expo-web-browser';
import { signInWithGoogleIdToken } from '../firebase/googleAuth';

WebBrowser.maybeCompleteAuthSession();

export function useGoogleSignIn(onError: (error: Error) => void) {
  const [request, response, promptAsync] = Google.useAuthRequest({
    clientId: process.env.EXPO_PUBLIC_GOOGLE_OAUTH_CLIENT_ID,
  });

  useEffect(() => {
    if (response?.type === 'success' && response.authentication?.idToken) {
      signInWithGoogleIdToken(response.authentication.idToken).catch(onError);
    } else if (response?.type === 'error') {
      onError(new Error('Google sign-in failed'));
    }
  }, [response]);

  return { promptAsync: () => promptAsync(), isReady: !!request };
}
```

```tsx
import { useEffect, useState } from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { onAuthStateChanged, User } from 'firebase/auth';
import { auth } from '../firebase/config';
import { SignInScreen } from '../screens/SignInScreen';
import { HomeScreen } from '../screens/HomeScreen';

const Stack = createNativeStackNavigator();

export function RootNavigator() {
  const [user, setUser] = useState<User | null>(null);
  const [initializing, setInitializing] = useState(true);

  useEffect(() => {
    return onAuthStateChanged(auth, (firebaseUser) => {
      setUser(firebaseUser);
      setInitializing(false);
    });
  }, []);

  if (initializing) {
    return null;
  }

  return (
    <NavigationContainer>
      <Stack.Navigator screenOptions={{ headerShown: false }}>
        {user ? (
          <Stack.Screen name="Home" component={HomeScreen} />
        ) : (
          <Stack.Screen name="SignIn" component={SignInScreen} />
        )}
      </Stack.Navigator>
    </NavigationContainer>
  );
}
```
